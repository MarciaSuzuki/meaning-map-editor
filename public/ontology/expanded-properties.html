Expanded Property Dimensions + Default Options Rule

Tripod Ontology v5.3 — Amendment for Meaning Map Editor
1 Universal Default Options Rule

Rule: Every field in the entire ontology carries two permanent options

not_specified — The text provides no basis for determination. The analyst chooses not to tag this field. This is the default state for every field when a clause is first loaded. Honest gaps are always preferable to guesses.

other — The analyst identifies something real in the text that does not match any listed value. When selected, the app opens a short text field for a label. All "other" entries are flagged for validator review. If a pattern of repeated "other" entries emerges across analysts, the ontology maintainer evaluates whether to add a new value to the list.

This rule applies uniformly to every enumerated field across all 14 layers. No exceptions. The app implements it as follows:

App behavior: Every dropdown, radio group, or selection field includes not_specified as the pre-selected default and other → as the final option. The "other" option expands an inline text input when selected. In the exported JSON, not_specified fields are included as "field": "not_specified" (not omitted), so the system can distinguish "analyst made no determination" from "field was never presented." Other entries export as "field": {"value": "other", "label": "analyst's description", "flagged": true}.

Section 2 below applies this rule to every layer in a compact reference. Section 3 presents the expanded Property Dimensions.

3 Expanded Property Dimensions

Below are the expanded dimensions. Values are selected for cross-linguistic universality — concepts that are lexicalized or grammaticalized in most of the world's languages. New values added in this expansion are marked with NEW.

Selection criteria for new values: Each addition was tested against three questions. (1) Is this concept expressed in unrelated language families? (2) Does it appear in biblical texts? (3) Would a community speech tagger need it in the Language Archive? Only concepts passing all three were added. Two entirely new dimensions were added: Sensory and Quantity/Size.

Every field across all 14 ontology layers now carries not_specified (default) and other → (flagged for review). This applies to ~45 enumerated fields with a combined ~350+ listed values. The "other" flag creates a feedback loop: if analysts repeatedly tag the same "other" value, it becomes a candidate for inclusion in the next ontology version.
